ENDPOINT API YANG HARUS DIIMPLEMENTASIKAN

Sistem: Tax Processing System (TPS) REST API
Base URL: http://localhost:8000
Autentikasi: Bearer Token (JWT) — diperoleh dari POST /auth/login

Keterangan Role
ADMINAkses penuh ke seluruh sistem
PETUGASInput data & lihat laporan, tidak bisa kelola user atau hapus data
WAJIB_PAJAKHanya melihat data miliknya sendiri
Akun Bawaan (Hasil Seeder) untuk Pengujian
Username Password Role & Keterangan
admin Pretest@2025 ADMIN — Akses penuh
petugas Petugas@2025 PETUGAS — Input data & laporan
budi Wajib@2025 WAJIB_PAJAK — Hanya lihat data sendiri

1. Auth

Login bersifat publik (tanpa token). Endpoint /auth/me dan /auth/logout memerlukan Bearer Token.

Method Endpoint Keterangan Token Body / Parameter
POST /auth/login Login dan dapatkan token JWT ❌ Tidak perlu { username, password } → { token, expires_in }
GET /auth/me Profil user yang sedang login ✅ Wajib —
POST /auth/logout Logout dan invalidasi token ✅ Wajib —

2. Role: ADMIN

Akses penuh — dapat mengelola user, wajib pajak, kewajiban, pembayaran, laporan, dan denda.

Method Endpoint Keterangan Body / Parameter
GET /users Daftar seluruh user (search & filter role) Query: search, role, per_page
GET /users/{id} Detail user —
POST /users Tambah user baru (ADMIN / PETUGAS) { name, username, email, password, role }
PUT /users/{id} Perbarui data user { name, username, email, password, role }
DELETE /users/{id} Hapus user (tidak bisa hapus akun sendiri) —
GET /wajib-pajak Daftar seluruh wajib pajak (search & filter) Query: search, jenis, status_aktif, per_page
GET /wajib-pajak/{id} Detail wajib pajak individu/badan usaha —
POST /wajib-pajak Tambah wajib pajak baru (sekaligus buat akun login) { jenis, nama, nik/nib, npwp, email, telepon, alamat, username, password }
PUT /wajib-pajak/{id} Perbarui data wajib pajak { nama, email, telepon, alamat, status_aktif, ... }
DELETE /wajib-pajak/{id} Hapus wajib pajak —
GET /kewajiban-pajak Daftar seluruh kewajiban pajak (filter wajib pajak & status) Query: wajib_pajak_id, jenis_pajak, status, per_page
GET /kewajiban-pajak/{id} Detail kewajiban beserta riwayat pembayaran —
POST /kewajiban-pajak Tambah kewajiban pajak baru { wajib_pajak_id, jenis_pajak, masa_pajak, pokok_pajak, jatuh_tempo }
POST /pembayaran Catat transaksi pembayaran (denda dihitung otomatis) { kewajiban_pajak_id, nominal, tanggal_bayar, keterangan }
GET /denda/{id} Detail denda berdasarkan ID pembayaran —
GET /laporan Laporan pembayaran per periode Query: tanggal_mulai, tanggal_akhir, jenis_pajak, status

4. Role: WAJIB_PAJAK

Hanya dapat melihat data miliknya sendiri. Akses ke data milik wajib pajak lain akan ditolak (403 Forbidden).

Method Endpoint Keterangan Body / Parameter
GET /wajib-pajak/me Data diri sendiri tanpa perlu ID (direkomendasikan) —
GET /wajib-pajak Daftar data diri sendiri (otomatis difilter, hasil paginasi) —
GET /kewajiban-pajak Daftar kewajiban pajak milik sendiri (otomatis difilter) Query: status, jenis_pajak
GET /kewajiban-pajak/{id} Detail kewajiban pajak (403 jika bukan miliknya) —
GET /denda/{id} Detail denda pembayaran (403 jika bukan miliknya) —
